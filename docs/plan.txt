
  # Player-LLM: Project Plan                                                  
                                                                              
  Fine-tune Qwen2.5-1.5B-Instruct to play poker in the style of each player   
  from the Pluribus dataset (10,000 six-handed NLH hands, Pluribus AI vs 5    
  pros).                                                                      
                                                                              
  ## Goals                                                                    
                                                                              
  • One model that plays convincing poker                                     
  • Swap a small LoRA adapter at runtime to imitate a specific player's style 
  • Runs on 8–16 GB GPU, CPU, or a tablet (via 4-bit quantization)

  ## Three-Stage Training Pipeline                                            
                                                                              
  ### Stage 0 — Domain-Adaptive Pretraining (DAPT)                            
                                                                              
  **What:** Run the standard next-token prediction objective on raw PHH action
  sequences from HandHQ. No instruction template. No hole cards needed.       
                                                                              
  **Why:** The base Qwen model has minimal poker exposure. This teaches it:   
                                                                              
  • Valid action syntax (f, cc, cbr, sm)                                      
  • Bet sizing distributions relative to pot and stacks                       
  • Street-by-street flow (preflop → flop → turn → river)                     
  • Stack-to-pot ratios that typically trigger shoves                         
                                                                              
  **Input format:** Serialized PHH fields as plain text, one hand per sample: 
                                                                              
    blinds=[50,100] stacks=[1200,980,1050,990,1010,970]                       
    p3 cbr 300 | p4 f | p5 cc | p6 f | p1 f | p2 cc                           
    board Jc7d2h | p2 cc | p3 cbr 200 | p2 f                                  
                                                                              
  **Volume:** 500K–1M hands from 400NLH+ stakes. Stop there; diminishing      
  returns.                                                                    
                                                                              
  **Script:** scripts/00_dapt.py                                              
  **Output checkpoint:** /data/models/qwen2.5-1.5b-instruct/dapt/             
                                                                              
  --------                                                                    
                                                                              
  ### Stage 1 — Supervised Fine-Tuning (SFT), All Players Joint               
                                                                              
  **What:** Instruction fine-tune on all 91K Pluribus decision points with    
  player                                                                      
  identity in the system prompt.                                              
                                                                              
  **Why joint (not separate per player):** Thin players (800–2,000 examples)  
  would overfit before learning poker. Joint training shares poker knowledge; 
  the player name conditions the style.                                       
                                                                              
  **Training method:** QLoRA (4-bit base + LoRA r=32, alpha=64) via           
  trl.SFTTrainer                                                              
                                                                              
  **Input format:** Chat template (see data-format.md for full spec)          
                                                                              
  **Script:** scripts/01_sft.py                                               
  **Output checkpoint:** /data/models/qwen2.5-1.5b-instruct/sft/              
                                                                              
  --------                                                                    
                                                                              
  ### Stage 2 — Per-Player LoRA Adapters                                      
                                                                              
  **What:** Starting from the Stage 1 checkpoint, fine-tune a separate small  
  LoRA                                                                        
  adapter for each player with sufficient data (>5K examples).                
                                                                              
  **Why:** Separates individual style from shared poker knowledge. Adapters   
  are                                                                         
  ~20–80 MB each and can be swapped at inference time without reloading the   
  base                                                                        
  model weights.                                                              
                                                                              
  **Training method:** LoRA r=16, alpha=32 (smaller than Stage 1 — we're      
  tuning                                                                      
  style, not knowledge)                                                       
                                                                              
  **Script:** scripts/02_lora_per_player.py                                   
  **Output:** /data/models/qwen2.5-1.5b-instruct/adapters/<player_name>/      
                                                                              
  --------                                                                    
                                                                              
  ## Model                                                                    
                                                                              
  **Base model:** Qwen/Qwen2.5-1.5B-Instruct                                  
  **Download location:** /data/models/qwen2.5-1.5b-instruct/base/             
  **Context window:** 32K tokens (a full poker hand is ~300 tokens)           
  **Quantization for inference:** 4-bit (Q4_K_M via bitsandbytes or llama.cpp 
  GGUF)                                                                       
                                                                              
  ### Why Qwen2.5-1.5B-Instruct                                               
                                                                              
  • Best quality/size ratio at 1.5B parameters                                
  • Strong structured reasoning despite small size                            
  • ~3 GB in fp16, ~1 GB in 4-bit — runs on tablet or CPU                     
  • Apache 2.0 license                                                        
  • Excellent instruction-following out of the box

  ## Evaluation                                                               
                                                                              
  • **Action accuracy:** % of held-out decisions where the model predicts the 
  correct action type (fold/call/raise) — target >60% (random baseline ~33%)  
  • **Bet size error:** Mean absolute error on raise amounts as fraction of   
  pot                                                                         
  • **Style divergence:** KL divergence between model's action distribution   
  and                                                                         
  target player's observed distribution per position and street               
                                                                              
  Hold out 10% of each player's hands for evaluation before training.
